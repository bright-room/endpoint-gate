# /plan スキル 使い方ガイド

## 概要

GitHub Issue を参照して実装プランを作成するスキル。

## 使い方

```
/plan 42        # Issue #42 の実装プランを作成
/plan 123       # Issue #123 の実装プランを作成
```

Issue 番号の指定が必須。

## 出力先

`.claude/outputs/plans/PLAN-<Issue番号>-<タイトル>.md` にファイル出力

## 処理の流れ

1. **Issue の取得** — `gh issue view` で Issue の要件を取得
2. **コードベースの調査**
   - `settings.gradle.kts` からモジュール構成を把握
   - `gradle-scripts/` の convention plugin からビルド・テスト構成を把握
   - 要件に最も近い既存モジュールをリファレンスとして特定
   - 影響範囲（core 層 → 非同期 adapter 層 → フレームワーク層の波及）を調査
3. **実装プランの作成** — 具体的な実装ステップ、テスト戦略を設計
4. **プランの出力** — ファイルとして出力

## 実装プランの構成

1. **概要** — 要件の要約と実装方針
2. **設計判断** — 複数アプローチのトレードオフ分析（必要な場合のみ）
3. **影響範囲** — 変更の影響があるモジュールのみ動的に列挙
4. **ファイル構成** — 変更・追加されるファイルのツリー表示
5. **実装ステップ** — Phase（論理的なまとまり）→ Step（具体的な作業単位）の階層構造
6. **ドキュメント更新** — README, CLAUDE.md, Auto-configuration imports, プロパティメタデータ等
7. **テスト戦略** — ソースセット別（test / integrationTest）のテスト方針
8. **確認事項** — Issue オーナーへの確認事項（必要な場合のみ）
9. **今後の展望** — スコープ外の関連拡張（必要な場合のみ）
